# 매출관리 AI 직원 — Commerce AI Company

전자상거래 사업자를 위한 **매출관리** 도구입니다. 단순히 매출액을 집계하는 것을 넘어,
① 취소/반품을 제외한 정확한 매출, ② 상품별 추세와 이상 탐지, ③ 광고비 대비
매출 효율(ROAS), ④ 원가를 반영한 실제 손익까지 함께 봐야 "잘 팔리고 있는지"가
아니라 "돈이 남는 장사인지"를 판단할 수 있다는 문제의식에서, 광고비와 원가를
매출 데이터와 별도 파일로 입력받아 같이 분석하는 기능을 포함합니다.

특히 광고비는 품목(상품)까지 매핑하면 **품목별로 광고를 켠 달과 끈 달의 평균
매출을 비교**해 "이 상품은 광고를 계속 돌릴 가치가 있는지, 꺼도 잘 팔리는
자연 판매 상품인지"를 월 단위로 보여줍니다.

## 플랫폼 탭 (쿠팡/네이버 등 소스별로 데이터 분리 관리)
화면 상단에 "전체 / 쿠팡 / 네이버 / + 플랫폼 추가" 탭이 있습니다. 각 탭은 서로 독립된
매출·광고비·원가 데이터를 갖고 있고, 브라우저에 저장되어 새로고침해도 유지됩니다.

- 탭을 눌러 전환하면 그 플랫폼에 업로드했던 데이터와 리포트가 그대로 나타납니다.
- "+ 플랫폼 추가"로 옥션, 11번가, 자사몰 등 원하는 이름의 탭을 계속 만들 수 있습니다.
- **"전체"** 탭은 모든 플랫폼 탭의 데이터를 합산해서 보여주는 계산 결과이며, 직접
  업로드는 할 수 없습니다(각 플랫폼 탭에서 업로드하세요). "🏆 상품별 매출 순위"와
  "📅 품목별 월별 상세"는 전체 탭에서도 그대로 동작하므로, 플랫폼을 합쳐서 품목 기준으로
  보고 싶을 때는 전체 탭을, 특정 플랫폼만 보고 싶을 때는 해당 탭을 선택하면 됩니다.
- 각 플랫폼 탭 옆의 👁/🚫 버튼은 **비활성화**입니다 — 데이터는 그대로 두고 "전체" 탭
  집계에서만 빼는 것으로, 탭 자체는 계속 눌러서 쓸 수 있습니다. × 버튼은 **삭제**로,
  그 플랫폼 탭과 저장된 매출·광고비·원가 데이터를 완전히 지웁니다(되돌릴 수 없어
  확인 후 진행). 최소 1개 플랫폼은 항상 남아 있어야 합니다.

## 데이터 검색 (키워드 칩) · 개별 데이터 ON/OFF·수정·삭제
"🔍 데이터 검색" 카드에 검색어를 입력하면 품목명·구매자명·날짜와 일치하는 항목이
아래 목록에 뜨고(↑/↓로 이동, Enter로 선택), 클릭(또는 Enter)하면 작은 칩으로 남아
결과를 필터링합니다. 칩은 다시 클릭하면 제거됩니다. 항목은 색으로만 구분됩니다
(품목=파랑, 구매자=초록, 날짜=노랑, 플랫폼=보라). 구매자명을 검색하려면 매출 CSV
매핑에서 "구매자명/회원명 컬럼"을 지정해야 합니다(선택 사항).

"전체" 탭에서는 각 매출 라인이 어느 플랫폼(쿠팡/네이버 등)에서 왔는지 결과 표에
"플랫폼" 열로 표시되고, 플랫폼 이름도 칩으로 선택해 필터링할 수 있습니다.

검색 결과 각 행에는 **ON/OFF · 수정 · 🗑 삭제** 버튼이 있습니다.

- **ON/OFF**는 그 매출 한 건을 통째로 켜고 끄는 버튼입니다. OFF로 꺼두면 삭제하지 않고도
  총 매출·상품 순위·품목별 월별 상세 등 **모든 계산에서 제외**됩니다(테스트성 주문이나
  의심스러운 건을 지우지 않고 잠시 빼둘 때 유용합니다). 다시 누르면 즉시 계산에 포함됩니다.
- **수정**을 누르면 카드 아래 "파일 없이 직접 입력해서 추가·수정" 폼에 그 값이 채워지고
  버튼이 "수정 저장"으로 바뀝니다. 값을 고치고 저장하면 원래 있던 데이터가 바로 갱신됩니다.
- **🗑**은 그 한 건을 완전히 삭제합니다(되돌릴 수 없음, 확인 후 진행).
- 검색창 위 **"전체 보기 / ON만 보기 / OFF만 보기"** 버튼을 누르면 검색어를 입력하지
  않아도 켜져 있는 데이터 또는 꺼둔 데이터만 목록으로 볼 수 있습니다.
- **"전체 데이터 삭제"** 버튼은 현재 플랫폼 탭의 매출·광고비·원가 데이터를 한 번에 모두
  지웁니다(플랫폼 탭 자체는 남아있음 — 탭까지 지우려면 상단 탭의 × 버튼을 쓰세요).

## 광고 ON/OFF 직접 지정
"📅 품목별 월별 상세"의 각 월별 행에는 "미지정 / ON / OFF" 버튼이 있습니다. 클릭할
때마다 순서대로 바뀌며, 실제 매출 데이터를 기준으로 ON인 달과 OFF인 달의 **평균
매출을 그대로 비교**해 보여줍니다(미래 변동을 예측하지 않고, 실제로 있었던 숫자만
사용합니다). 광고비 CSV로 자동 판별된 상태 위에 직접 지정한 값이 우선 적용되므로,
광고비 데이터가 없는 품목도 이 버튼만으로 ON/OFF 비교를 만들 수 있습니다.

## 실행 방법 (설치 없이 체험)
`index.html`을 더블클릭해서 브라우저로 열고 매출 카드의 "샘플 데이터"를 누르면
바로 전체 흐름(매출 추이 → Top 상품 → 이상 탐지 → 코멘트 → 리포트 인수인계)을
확인할 수 있습니다. 이어서 "샘플 광고비 데이터로 체험하기" · "샘플 원가 데이터로
체험하기"까지 누르면 ROAS · 원가 반영 손익 · **품목별 월별 광고 ON/OFF 비교**까지
함께 계산됩니다(샘플 데이터는 "냉감 이불커버 세트"가 6월엔 광고 ON, 7월엔 광고
OFF인 상황을 재현합니다). 데이터는 브라우저 localStorage에만 저장됩니다.

## 실제 내 쇼핑몰 데이터로 쓰는 방법
1. 쿠팡 Wing → 정산 → 매출내역/정산내역 다운로드 (CSV 또는 엑셀)
2. index.html에서 파일 업로드
3. 열 매핑(날짜/상품명/매출액) 확인 후 "데이터 적용"

같은 폴더의 `쿠팡주문.csv`는 실제 쿠팡 주문 다운로드 파일과 같은 구조(주문일자, 주문번호,
구매자ID, 상품명, 옵션, 수량, 판매가, 주문상태)의 샘플입니다. 이 구조에서 중요한 점:

- **주문상태에 "취소", "반품"이 섞여 있습니다.** 이 건들을 매출로 그대로 합산하면 실제보다
  매출이 부풀려집니다. 그래서 열 매핑 화면에 **"주문상태 컬럼"**과 **"매출에서 제외할 상태값"**
  (기본값: 취소, 반품, 환불)을 추가했습니다. 컬럼을 지정하면 해당 건은 자동으로 매출 집계에서
  빠지며, 몇 건을 제외했는지 리포트에도 표시합니다.
- 이 샘플의 "판매가"는 옵션당 개별 단가가 아니라 **해당 주문 라인의 합계 금액**이라 별도
  단가×수량 계산 없이 매출액 컬럼으로 바로 매핑하면 됩니다. (다른 사장님의 파일은 판매가가
  단가로 되어있을 수 있으니, 그 경우엔 수량 컬럼을 지정해 실제 매출과 맞는지 한 번 검산해보세요.)

#### CSV든 엑셀이든, 컬럼 순서가 달라도 괜찮습니다
매출·광고비·원가 세 카드 모두 **CSV와 엑셀(.xlsx)** 파일을 그대로 올릴 수 있습니다.
엑셀 파싱은 외부 라이브러리 없이 브라우저 내장 기능만으로 직접 구현했습니다 — 별도
설치나 인터넷 연결 없이 로컬에서 동작합니다. 다만 구버전 `.xls`(2007 이전 엑셀
형식)는 지원하지 않으니, 그 경우 엑셀에서 "다른 이름으로 저장 → .xlsx 또는 CSV"로
바꿔서 올려주세요. 대용량 파일도 문제없이 처리되지만, 브라우저 안에서 파싱하는
구조라 파일이 아주 크면(수만 행 이상) 다소 시간이 걸릴 수 있습니다.

컬럼 순서는 파일마다 달라도 상관없습니다 — "매출액, 날짜, 이름" 순으로 되어 있든
"날짜, 이름, 매출액" 순으로 되어 있든, 매핑 화면에서 어떤 컬럼이 어떤 역할인지
지정하기만 하면 내부적으로는 항상 같은 순서(날짜·상품명·매출액·수량·구매자 등)로
정리해서 저장합니다.

#### 파일을 고칠 필요는 없습니다 — 포맷 프리셋으로 컬럼만 골라두면 끝 (API 키 불필요)
쿠팡, 네이버 스마트스토어 등 플랫폼마다 다운로드 파일의 컬럼 이름과 순서가 다릅니다.
**엑셀 파일 자체를 수정할 필요는 전혀 없습니다.** 업로드하면 실제 파일의 헤더가 그대로
드롭다운에 나열되고, "이 컬럼이 날짜예요/매출액이에요"라고 골라주기만 하면 됩니다. 이
매핑 작업은 전부 브라우저 안에서 끝나며, 외부 AI API나 인터넷 연결이 전혀 필요 없습니다.

매번 고르기 번거롭다면 **"포맷 프리셋"**을 쓰세요. 매출/광고비/원가 매핑 화면마다 프리셋
드롭다운이 있고, 여기서 플랫폼을 선택하면 저장된 컬럼명과 실제 파일의 헤더가 일치하는
항목을 자동으로 찾아 매핑해줍니다. 일치하지 않는 항목은 자동으로 채워지지 않을 뿐(잘못된
값이 채워지지는 않습니다) 직접 골라서 "현재 매핑을 프리셋으로 저장" 버튼으로 새로 저장하면
다음부터는 그 이름으로 재사용할 수 있습니다. 더 이상 쓰지 않는 프리셋은 드롭다운에서
선택한 뒤 옆의 **🗑 버튼**으로 삭제할 수 있습니다(기본 제공 프리셋은 삭제할 수 없습니다).

- 매출 매핑에는 **"쿠팡 (Wing 주문/매출내역)"** 프리셋이 기본 내장되어 있고, 이 프로젝트의
  `쿠팡주문.csv` 실제 구조로 검증된 값입니다.
- **"네이버 스마트스토어 (주문 다운로드, 추정)"** 프리셋도 함께 제공하지만, 네이버 다운로드
  양식은 매장마다·시점마다 컬럼명이 조금씩 다를 수 있어 일반적인 추정값입니다.
- 광고비 매핑에는 **"쿠팡 광고관리자 (추정)" · "네이버 검색광고 (추정)" · "메타 광고관리자
  (추정)"** 프리셋을, 원가 매핑에는 흔히 쓰는 컬럼명 조합으로 **"기본(상품명·원가)" ·
  "매입가 기준" · "제조원가 기준"** 프리셋을 기본 제공합니다. 모두 실제 다운로드 파일마다
  컬럼명이 조금씩 다를 수 있는 "추정값"이므로, 헤더가 다르면 자동으로 채워지지 않을 뿐
  잘못된 값이 채워지지는 않습니다. 직접 한 번 매핑한 뒤 원하는 이름으로 새로 저장해두면
  다음부터는 그 프리셋을 바로 선택해서 쓸 수 있습니다.

#### 샘플 양식 다운로드
어떤 컬럼으로 파일을 준비해야 할지 감이 안 잡히면, 각 업로드 카드의
**"샘플 양식 다운로드"** 버튼으로 우리가 쓰는 컬럼 구성이 담긴 빈 CSV(예시 행 1개 포함)를
내려받아 참고하거나, 그 파일에 직접 데이터를 채워 그대로 업로드해도 됩니다.

#### 파일 없이 직접 입력해서 추가·수정, 불러온 데이터 보기·삭제
데이터가 몇 건뿐이라 파일을 만들 필요가 없다면, 각 업로드 카드 아래의
**"파일 없이 직접 입력해서 추가·수정"**을 펼쳐 값을 채우고 "추가" 버튼을 누르면 한 건씩
바로 리포트에 반영됩니다(매출: 날짜·상품명·매출액·수량·구매자, 광고비: 날짜·광고비·채널·
상품명, 원가: 상품명·원가). 파일 업로드와 자유롭게 섞어 써도 됩니다.

광고비·원가 카드에는 **"불러온 광고비/원가 데이터 보기·수정·삭제 (N건)"**을 펼치면 지금까지
쌓인 데이터가 표로 나옵니다. 행마다 "수정"을 누르면 위 입력 폼에 값이 채워지고 저장 버튼이
"수정 저장"으로 바뀌며, 🗑로 그 한 건만 삭제할 수 있습니다(파일로 올렸든 직접 입력했든
동일하게 적용됩니다). 매출 데이터는 "🔍 데이터 검색" 결과 목록에서 같은 방식으로 수정·
삭제·ON/OFF를 할 수 있습니다.

#### 광고비는 별도 파일로 (선택, ROAS·품목별 ON/OFF 비교용)
매출 데이터와 별개로 "📢 광고비 데이터 불러오기" 카드에 **날짜 / 광고비 / 채널(선택) /
상품명(선택)** 컬럼이 있는 CSV를 올릴 수 있습니다. 같은 폴더의 `샘플_광고비데이터.csv`가
예시입니다. 매출 데이터를 먼저 불러온 뒤 광고비를 적용하면 아래가 자동으로 계산됩니다.

- 광고비 합계, ROAS(매출÷광고비), 광고비율(광고비÷매출), 광고비 차감 순매출
- ROAS가 1배 미만이거나 직전 기간 대비 크게 떨어졌을 때 경고 표시
- 광고비는 늘었는데 매출 증가가 그에 못 미칠 때 경고 표시

**상품명 컬럼까지 매핑하면** "📅 품목별 월별 상세" 카드에서 품목별로 광고를 켠 달과 끈 달을
구분해 평균 매출을 비교합니다. 광고비 CSV에 아예 등장하지 않는 상품은 "광고 없음(자연
판매)"로 표시됩니다 — 모든 상품에 광고비가 있을 필요는 없습니다. 상품명 컬럼을 지정하지
않으면 전체 합계 ROAS만 계산되고, 품목별 비교는 생략됩니다.

매출·광고비 두 파일의 날짜 형식이 다르면(예: `2026.6.7` vs `2026-06-07`) 자동으로 같은
형식(YYYY-MM-DD)으로 맞춰 집계합니다.

#### 원가도 별도 파일로 (선택, 손익 계산용)
"🏷️ 원가 데이터 불러오기" 카드에 **상품명 / 개당 원가** 두 컬럼만 있는 CSV를 올릴 수
있습니다. 같은 폴더의 `샘플_원가데이터.csv`가 예시입니다. 쿠팡 정산 파일에는 원가 정보가
없기 때문에, 사장님이 직접 관리하는 매입가/제조원가 표를 그대로 올리면 됩니다. 원가를
적용하면:

- 총 원가, 매출총이익(매출-원가), 영업이익(매출총이익-광고비), 영업이익률이 계산됩니다.
- 매출 데이터에는 있지만 원가 파일에 없는 상품은 원가 0원으로 계산되며, 리포트에 몇 개
  품목이 빠졌는지 경고로 표시됩니다(이익을 몰래 부풀리지 않기 위함입니다).

#### 상품별 매출 순위 (전체, 더보기로 펼쳐보기)
"🏆 상품별 매출 순위" 카드는 상위 10개만 보여주지 않고 **전체 품목을 순위대로** 계산합니다.
처음엔 10개만 보이고 "더보기"를 누르면 나머지가 모두 펼쳐집니다. Markdown 리포트에도 전체
순위가 그대로 나갑니다.

#### 품목별 월별 상세 (전체 품목, 펼쳐보기)
매출 데이터만 있어도 "📅 품목별 월별 상세" 카드에서 **순위와 무관하게 전체 품목**을 월별로
펼쳐볼 수 있습니다. 각 품목을 펼치면 월별 매출·원가·광고비·순이익·광고 ON/OFF 상태가
표에 나오고, 광고 ON 달과 OFF 달이 모두 있는 품목은 두 구간의 평균 매출을 자동으로
비교해줍니다.

## 빠른 체험 가이드
1. 상단 탭에서 "쿠팡" 탭을 선택한 상태로 매출 카드의 "샘플 데이터" 클릭 → 최근 30일
   매출 추이 그래프 확인
2. 상품 순위 중 "휴대용 목선풍기"가 최근 급감(-대략 70%)으로 이상 탐지에 뜨는 것 확인.
   "여름 반팔티"는 반대로 최근 급증하는 트렌드 포착 사례로 함께 확인 가능
3. 광고비·원가 카드의 "샘플 데이터"도 각각 클릭 → ROAS · 매출총이익 · 영업이익이 함께
   계산됨
4. "📅 품목별 월별 상세"에서 "냉감 이불커버 세트"를 펼쳐 6월(광고 ON)과 7월(광고 OFF)의
   평균 매출 차이 확인 — 광고를 계속 돌릴 가치가 있는지 판단하는 근거가 됩니다. 다른
   달의 ON/OFF 버튼을 직접 눌러 비교가 바뀌는 것도 확인해보세요
5. "🔍 데이터 검색"에 상품명이나 구매자명을 입력해 필터링 결과 확인. 아무 행이나 OFF로
   꺼보고 위쪽 매출 KPI가 바로 줄어드는 것을 확인해보세요(다시 ON을 누르면 원복됩니다)
6. "네이버" 탭으로 전환해 별도 데이터를 올리고, "전체" 탭에서 두 플랫폼 합산 결과 확인
7. handoff.json / Markdown 리포트 내보내기로 회의 자료·인수인계 문서를 바로 생성
