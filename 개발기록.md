## 1. 프로젝트 구조

- **형태**: 프레임워크 없는 단일 `index.html` (HTML + CSS + 순수 JS). 서버·DB 없음.
- **데이터 저장**: 브라우저 localStorage (`salesai*` 접두사 키). 플랫폼 탭별로
  `salesai_platform_data_<id>`에 매출 레코드·파일 목록 저장.
- **GitHub**: `leechungchung/sales-ai-employee` (공개). 로컬 폴더에는 `.git`이 없어서
  GitHub Contents API(`gh api -X PUT .../contents/index.html`)로 커밋을 올린다.
- **Vercel**: 리포지토리에 연결되어 있어 **GitHub에 커밋이 올라가면 자동으로 Production 배포**.
  - 배포 주소(고정): https://sales-ai-employee.vercel.app
- **레코드 구조**: `{date, product, amount(합계), unitPrice(개당 단가|null), qty, customer,
  cost(개당 원가|null), adAmount, adChannel, included, fileId, platformName?}`

## 3. 기능 (2026-07-22 추가/개편분)

- **매출액 vs 개당 단가**: 컬럼 매핑·수기 입력에 [합계 금액 | 개당 단가] 토글.
  개당 단가 선택 시 단가 × 수량으로 합계 환산 저장, `unitPrice` 별도 보관.
  컬럼명에 "단가" 포함 시 자동 전환. 프리셋에 선택값 저장.
- **데이터 검색**: 카드 제목 옆 상시 검색칸(기본 접힘, 입력 시 자동 펼침), 결과 5줄 + 더보기,
  ON/OFF 필터는 결과 헤더 우측. 표에 단가·수량 열. 수정 모드에서 단가·수량·합계 연동
  (단가/수량 수정 → 합계 자동 계산, 합계 직접 수정 → 단가 연동 해제).
- **파일 재매핑**: 업로드 시 원본 표(5천 행 이하)를 localStorage에 저장 →
  **새로고침 후에도 파일명 클릭으로 컬럼 재매핑 가능**. 용량 초과 시 raw만 생략.
- **수량 전면 반영**: KPI(총 판매 수량), 상품 순위(개수 병기), 월별 상세(수량 열),
  코멘트·Markdown·handoff 리포트.
- **광고 ON/OFF**: "미지정" 순환 버튼 제거 → ON | OFF 두 버튼. 광고비 데이터 없는 달은
  **기본 ON**(사용자는 끈 달만 OFF 지정). 지정 해제는 켜진 버튼 재클릭.
- **💾 전체 백업/복원**: `salesai*` 키 전체를 JSON 하나로 내보내고 복원(덮어쓰기 확인 후 reload).
- **🎯 월 매출 목표**: 플랫폼별 목표 저장(전체 탭은 합산). 달성률 게이지 + 월말 페이스 예측
  (데이터 마지막 달의 경과일 기준). 페이스 부족 시 게이지 주황. 인라인 편집(Enter/Esc).
- **📌 오늘의 판단** (핵심 기능): 데이터를 행동 제안으로 변환, 금액 임팩트 순 최대 3개.
  - 역마진: 원가·광고비 반영 영업이익 < 0 → 가격/원가 재검토
  - 광고 손익분기: 상품별 ROAS < 1÷마진율 → 광고 중단 검토 + 월 절감액
  - 자연판매: 광고 쉰 달 평균 ≥ 광고한 달의 80% → 광고비 절감 제안
  - 급성장/급감: 최근 3일 vs 이전 7일 평균 ±80%/-40% → 재고 확보/원인 확인
  - 각 판단 클릭 → 계산 근거 표 펼침. **결정적 근거 행에 "핵심" 배지 + 인디고 강조**.
    펼친 상태에서는 안내 툴팁 제거. "○○ 데이터 보기" 버튼 → 검색 카드로 필터 이동.
  - 같은 판정이 상품 순위·월별 상세에 배지(⚠ 역마진, 💸 광고 재검토, 🌱 자연판매,
    🎯 광고 효자, 🚀/📉)로 표시. 배지 툴팁 = 의미 1줄 + 데이터 1줄.
  - 데이터 부족 시 힌트("원가를 입력하면 손해 보는 상품까지 찾아드려요.").
- **👥 재구매 고객**: 구매자명 기반 재구매율·재구매 매출 비중·VIP 10명. 데이터 없으면 카드 숨김.
- **📆 요일별 매출 패턴**: 하루 평균 기준 요일 비교 + 최강 요일 코멘트. 7일 미만이면 숨김.
- **이상 징후 탐지 카드 삭제**: 오늘의 판단·배지와 중복이라 제거(계산 로직은 유지).
- **월별 상세 카드**: 기본 접힘, 접힘 버튼 라벨 "보기".

## 4. 카피(UX 라이팅) 규칙 — 반드시 지킬 것

사용자가 정한 규칙. 새 문구를 추가할 때도 그대로 따른다.

1. **토스 스타일**: 짧게 끊고, 해요체로, 숫자가 주인공. 전문용어 대신 돈으로 환산해서 말한다.
   - 예: "ROAS 1.76배" (X) → "광고비 1만원당 매출 17,600원" (O). ROAS는 괄호 병기만.
2. **한 문장이 끝나면 줄바꿈**: `sentenceBreak()` 헬퍼가 온점 뒤 공백을 `<br>`로 변환.
   툴팁은 표시 시점에 같은 규칙 자동 적용(`white-space:pre-line`).
3. **온점 규칙**: 말 문장(~해요/~예요)에는 온점을 붙인다. 숫자·데이터 줄
   ("누적 -152,472원", "1만원 → 95원")에는 붙이지 않는다.
4. **줄바꿈 단위**: 한글은 어절 단위로만 꺾인다(body 전체 `word-break:keep-all`).
5. **툴팁 데이터 분리**: 배지 툴팁은 "1줄 = 의미, 2줄 = 데이터" 구조.
6. 툴팁 글자 크기는 본문 수준(13.5px) 이상 유지.
